## 2025-02-15 - Prevent SSRF via MPRIS Album Art URLs
**Vulnerability:** The MPRIS module fetches album art URLs directly from untrusted metadata broadcast by any local media player (or remote players via MPRIS proxy). It blindly passes the URL to `reqwest`, allowing Server-Side Request Forgery (SSRF) where a malicious player can force the application to hit internal, private, or loopback network addresses.
**Learning:** Even local IPC metadata from "trusted" desktop protocols like MPRIS must be treated as untrusted user input when it contains URIs that trigger network or filesystem I/O. Furthermore, DNS resolution must be performed manually to block malicious IPs, as DNS rebinding (TOCTOU) can bypass simple URL host string checks.
**Prevention:** Always manually resolve external URIs, validate that the resulting IP is public/safe (blocking loopback, private, link-local, broadcast, etc.), and use `reqwest`'s `.resolve()` builder method to pin the host to the validated IP to prevent DNS rebinding. Additionally, disable redirects (`redirect::Policy::none()`) to prevent redirect-based bypasses. Avoid unstable features like `Ipv4Addr::is_documentation()` when validating IPs in stable Rust.
