## 2025-02-28 - Reqwest `.text().await` Memory Exhaustion (OOM)
**Vulnerability:** The engine downloaded GitHub release files (like `SHA256SUMS.txt` and signatures) using `reqwest::Response::text().await`. If a malicious actor compromised the release endpoint or hijacked DNS to serve a massive text payload, `.text().await` would buffer it entirely in memory, leading to an Out-of-Memory (OOM) Denial-of-Service (DoS) crash of the application.
**Learning:** Even for "trusted" infrastructure like GitHub releases, relying on boundless buffer reading functions (`.text().await`, `.json().await`, `.bytes().await`) is a security risk. All untrusted or remote payloads must have strict size limits applied during reading.
**Prevention:** Always use the engine's `cosmic_wallpaper::modules::utils::read_capped` function with an appropriate `MAX_SIZE` (e.g. `1024 * 1024` for text files) instead of `reqwest`'s unbound convenience methods. For text, decode the capped bytes manually using `String::from_utf8`.
