## 12-02-2025- Rust Disjoint Field Borrowing Saves Hot Path Heap Allocations
**Learning:** The codebase previously cloned the font family `String` on every frame because of a stale comment warning about borrow checker conflicts. After helper functions like `push_text_buffer` were refactored to use disjoint field borrowing (passing specific mut fields like `&mut renderer.font_system` rather than `&mut renderer` itself), the borrow checker became fully capable of borrowing the font family `&str` directly from the config without cloning, but the stale cloning remained on the hot rendering path.
Additionally, calling `get_pixel` on a `DynamicImage` enum within the hot palette extraction loop caused dynamic dispatch on every pixel; extracting `RgbaImage` once before the loop bypassed this entirely.
**Action:** Always verify if borrow-checker workaround comments (like "cloned to avoid borrow issues") are still accurate in the current architecture. If helper functions have been refactored to disjoint mutable field borrows, we can safely replace allocations/clones with zero-copy references on the hot path. Also, always resolve `DynamicImage` to a concrete typed reference like `RgbaImage` before nested pixel loops.
