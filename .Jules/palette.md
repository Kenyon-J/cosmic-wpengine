## 2024-05-18 - Editor Dirty State Tracking without Allocating Strings
**Learning:** In `cosmic::iced` applications with built-in text editors, track a 'dirty' state to conditionally enable the 'Save' button, append a visual indicator (like `*`), and provide disabled tooltips. To prevent severe performance regressions, NEVER call `editor_content.text()` inside the `view` function as it allocates a new String every frame. Instead, compute and store an `is_dirty: bool` during text modification messages. Note that `EditorAction` encompasses non-modifying actions (cursor movement, clicking), so explicitly match `if let cosmic::widget::text_editor::Action::Edit(_) = action` when setting the dirty flag to avoid false positives.
**Action:** When working on text editor widgets in iced/cosmic apps, add an explicit boolean `is_dirty` to the state to enable/disable UI actions without checking content each frame in `view()`.
